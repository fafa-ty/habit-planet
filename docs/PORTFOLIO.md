## 一句话

---

## 背景 & 痛点

- 现有习惯 App：Habitica 学习成本高，Done 仅 iOS，小日常重颜值轻数据
- 目标用户：18–35 岁，有自我提升意愿但缺少轻量、跨平台的记录工具

---

## 核心设计决策

1. **3 秒打卡**：单击卡片即完成，不做多步流程  
2. **本地优先**：零注册冷启动，可选云同步  
3. **数据可视化**：GitHub 风格热力图 + 周报，强化正反馈  
4. **渐进扩展**：MVP → V1.1（提醒/归档/同步）→ V2（PWA/社交）

# 面试演示脚本（10 分钟版）

## 结构


---

## 1. Hook — 为什么做（1 分钟）

**话术：**

> 「我想解决一个很具体的问题：我自己用过 Habitica，功能很强，但每天要理解 RPG 规则，打卡要 15 秒以上，最后放弃了。
>
> 我调研后发现，轻量、跨平台、数据可见的习惯工具有缺口。所以我做了习惯星球——**打开就能用，3 秒打卡，热力图看见进步**。」

**展示：** 在线 Demo 首页

---

## 2. 竞品 — 为什么这样设计（2 分钟）

**话术：**

> 「我系统对比了 Habitica、小日常、Done、Loop 四款产品。」

**展示：** `docs/COMPETITOR_ANALYSIS.md` 功能矩阵

| 我们的选择 | 原因 |
|-----------|------|
| 不做 RPG | Habitica 验证过强，但门槛高 |
| 做热力图 | Done 验证有效，我们补跨平台 |
| Web 优先 | 小日常/Done 无 Web，降低使用门槛 |
| 开源免费 | 差异化，方便二次开发 |

**金句：** 「比 Habitica 轻，比 Done 广，比小日常更数据向。」

## 常见面试追问 & 参考答案

**Q：为什么不做原生 App？**  
A：Web + PWA 覆盖 iOS/Android/桌面，迭代快。V2 可考虑 Capacitor 套壳。

**Q：云同步怎么保证冲突？**  
A：按 habitId+date 合并 checkIn，取较新 completedAt；habit 按 archivedAt 取新。

**Q：下一步做什么？**  
A：PWA 离线、每周 N 次灵活频率、好友 Streak 排行榜（不做 RPG）。

**Q：最大的设计 trade-off？**  
A：放弃游戏化换低门槛；用 confetti + 成就补激励，数据不够 RPG 强但 3 秒打卡是核心。

**Q：如果只有一周还会砍什么？**  
A：砍云同步和 React 版，只留 MVP：打卡 + Streak + 本地存储。
