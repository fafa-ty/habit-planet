# 习惯星球 HabitPlanet — 产品设计文档

> 一份面向产品新手的完整 PRD 范例。你可以照着这个结构，设计任何一款小型 App。

---

## 1. 产品概述

### 1.1 一句话描述
**习惯星球** 是一款帮助用户养成每日好习惯、可视化坚持进度的个人习惯追踪 Web App。

### 1.2 为什么选这个题目？（新手友好）
| 维度 | 说明 |
|------|------|
| 用户痛点清晰 | 人人都想坚持运动/阅读/早睡，但容易放弃 |
| 功能边界明确 | 创建习惯 → 每日打卡 → 看进度，逻辑简单 |
| 数据模型简单 | 习惯 + 打卡记录，两个核心实体 |
| 可渐进扩展 | 先做 MVP，再加统计、成就、提醒 |
| 无需后端 | 本地存储即可跑通完整体验 |

### 1.3 目标用户
- **主要用户**：18–35 岁，有自我提升意愿的学生/职场新人
- **使用场景**：早晨起床后勾选今日习惯；睡前回顾完成情况

### 1.4 产品目标
- 让用户 **3 秒内** 完成今日打卡
- 用 **连续天数（Streak）** 激励持续坚持
- 用 **可视化图表** 让进步"看得见"

## 2. 用户故事（User Stories）

```
作为 一个想养成阅读习惯的人
我希望 每天勾选"阅读 30 分钟"
以便 追踪自己是否坚持，看到连续打卡天数

作为 一个健身新手
我希望 看到本周完成率的热力图
以便 知道哪几天偷懒了，及时调整

作为 第一次使用的用户
我希望 有引导流程帮我创建第一个习惯
以便 不用研究界面就能开始
```

## 4. 信息架构

```
习惯星球
├── 今日（首页）      ← 打卡主战场
├── 习惯管理          ← CRUD + 分类筛选
├── 数据统计          ← 热力图 + 图表 + 成就
└── 设置              ← 主题 / 导出 / 重置
```

## 5. 数据模型

### 5.1 Habit（习惯）
```json
{
  "id": "uuid",
  "name": "阅读 30 分钟",
  "icon": "📚",
  "color": "#6366f1",
  "category": "study",
  "createdAt": "2026-06-01",
  "archived": false
}
```

### 5.2 CheckIn（打卡记录）
```json
{
  "habitId": "uuid",
  "date": "2026-06-04",
  "completedAt": "2026-06-04T08:30:00.000Z"
}
```

### 5.3 派生数据（前端计算）
- `streak`：从今天/昨天往回数连续打卡天数
- `completionRate`：某时间段内 完成天数 / 总天数
- `achievements`：根据 streak、总打卡数等条件解锁

## 7. 交互设计要点

| 交互 | 设计决策 | 原因 |
|------|----------|------|
| 打卡 | 单击卡片即可切换 | 减少操作步骤 |
| 删除 | 长按 / 编辑页确认 | 防误删 |
| 新建习惯 | 模态弹窗 + 图标选择器 | 不离开当前上下文 |
| 完成全部 | confetti 纸屑动画 | 正向反馈，增强动力 |
| 空状态 | 引导创建第一个习惯 | 降低冷启动流失 |

## 9. 成功指标（假如上线）

| 指标 | 目标 |
|------|------|
| 首日创建习惯数 | ≥ 2 |
| 7 日留存 | ≥ 30% |
| 日均打卡次数 | ≥ 2 |

## 10. 新手设计 checklist

设计任何产品时，按这个顺序思考：

1. **谁在用？解决什么问题？** → 用户 + 痛点
2. **最少要几个功能才能用？** → MVP
3. **数据存什么？** → 数据模型
4. **有几个页面？怎么跳转？** → 信息架构
5. **每个页面放什么？** → 线框图
6. **关键操作怎么走？** → 交互流程
7. **长什么样？** → 视觉规范
8. **怎么知道做成功了？** → 成功指标
